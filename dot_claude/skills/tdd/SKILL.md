# Test-Driven Development (TDD)

テスト駆動開発のワークフローに従って実装を進める。

## 基本サイクル

1. **テストを書く** - 期待される入出力に基づき、まずテストを作成する
2. **テストを実行** - テストが失敗することを確認する
3. **コミット** - テストが正しいことを確認できた段階でコミットする
4. **実装** - テストをパスさせる最小限の実装を行う
5. **リファクタリング** - テストがパスした状態でコードを改善する
6. **繰り返し** - すべてのテストが通過するまで繰り返す

## ルール

- 実装コードより先にテストを書く
- 実装中はテストを変更しない（テストを修正するのは設計が間違っていた場合のみ）
- 1つのテストが通ったら次のテストに進む
- テストケースは日本語で記述する

## 注意事項

- テストは独立して実行できるようにする
- モックは必要最小限に留める
- テストの可読性を重視する
