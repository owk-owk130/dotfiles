# 見積もりガイドライン・覚え書きルール

## 工数 vs 売値工数

- **工数**: 実際にかかる作業時間（スターターキット・LP学園・他案件のコード流用時は、カスタマイズにかかる時間のみ）
- **売値工数**: 本来の実装工数を反映した値。流用がなければゼロから作る場合の工数
- スターターキット利用時は「工数 < 売値工数」となる
- 売値工数がない（流用なし）場合は工数と同値にする

## 工数の基準

- **中堅プレイヤー（2〜3年目）** を想定した工数
- シニアなら0.7掛け、ジュニアなら1.3〜1.5掛けが目安

## 小項目の分け方

ページごとに以下の作業種別で分ける:

1. **マークアップ** - HTML/CSSコーディング（レスポンシブ含む）
2. **◯◯機能実装** - JavaScript/フレームワークによる機能実装（◯◯には具体名を入れる）
3. **API結合** - バックエンドAPIとの連携実装
4. **演出** - アニメーション、トランジション、スクロール演出等

## 見積もり精度レベル

| レベル | 振れ幅 | 用途 |
|--------|--------|------|
| 超概算 | -50%〜+100% | 初回相談、予算感確認 |
| 概算 | -25%〜+50% | 企画確定前、要件が曖昧 |
| 確定 | -5%〜+10% | デザイン・仕様FIX後 |

- 備考に精度レベルを必ず明記する
- プロデューサーに必要な精度を事前確認する

## 忘れがちな項目チェックリスト

見積もり作成後、以下が漏れていないか確認する:

- [ ] Analytics対応（GA4、GTM、イベント設計）
- [ ] SNSシェア・OGP設定
- [ ] 多言語対応
- [ ] 他社サーバー最適化（画像圧縮、CDN設定等）
- [ ] SEO（meta設定、構造化データ）
- [ ] ファビコン・Webクリップアイコン
- [ ] 404ページ
- [ ] ローディング・スケルトン
- [ ] エラーハンドリング（通信エラー、タイムアウト）
- [ ] ブラウザ対応範囲の確認
- [ ] アクセシビリティ対応
- [ ] パフォーマンス最適化（Core Web Vitals）

## 備考の書き方

- **備考（プロジェクト固有）**: そのプロジェクト特有の判断理由を記載
  - 例: 「企画が固まってないので超概算」「前回のものを使う」「デザイン未確定のため+バッファ」
- **参考情報（汎用）**: テンプレートの説明文。どんな案件でも使える一般的な情報
  - 例: 「PC 8h + SP 8h が基準」「エンドポイント数×2-4h」
