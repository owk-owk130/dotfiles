# フロントエンド工数見積もりスキル

フロントエンド/マークアップ案件の工数見積もりを対話的に作成する。
以下の4フェーズで進行する。

## Phase 1: ヒアリング

ユーザーの最初のメッセージに情報が含まれている場合は、不足分のみ質問する。
一度に全部聞かず、優先度の高いものから2〜3項目ずつ聞く。

### 質問項目（優先度順）

1. **プロジェクト概要** - 何のサイト？目的は？
2. **ページ構成** - TOP有無、主要ページとその役割
3. **見積もり精度** - 超概算(-50%〜+100%) / 概算(-25%〜+50%) / 確定(-5%〜+10%)
4. **技術スタック** - HTML/CSS/JS / React / Next.js / Vue / Nuxt / Astro
5. **スターターキット・既存コード流用の有無**
6. **主な機能要件**:
   - アニメーション演出の有無・リッチさ
   - API連携（外部API、自社API）
   - 画像生成・Canvas操作
   - フォーム・入力系
   - SNSシェア・OGP
   - 外部サービス連携（Apps Script、CMS等）
7. **レスポンシブ対応範囲** - SP+PC / SP+タブレット+PC
8. **SEO要件、Analytics要件**
9. **納品形態** - ファイル納品 / デプロイ / ソースコード納品
10. **作成者名、確認者名**（CSVメタ情報用）

### ヒアリング完了後

収集した情報をサマリーとしてマークダウンで表示し、ユーザーに確認を求める。

## Phase 2: 項目生成

`references/template.md` を読み込み、ヒアリング結果に基づいて見積もり項目と工数を生成する。

### 生成ルール

- **中項目はページ単位**で作る（TOP / 〇〇ページ / △△画面）
- **小項目は作業種別**で分ける（マークアップ / 機能実装 / API結合 / 演出）
- 工数はテンプレートのベースライン値をページ複雑度で調整
- 売値工数はスターターキット利用有無で判断（流用なしなら工数と同値）
- **QA工数** = 実装工数合計の **20〜30%** で自動算出（複雑な案件は30%寄り）
- **進行管理** = 全体の **3〜10%** で自動算出（長期案件は10%寄り）
- **備考にはプロジェクト固有の判断理由**を記載する

### `references/estimation-guidelines.md` を読み込み、以下を確認:
- 忘れがちな項目チェックリストに照らして漏れがないか
- 精度レベルが備考に明記されているか
- 工数 vs 売値工数の使い分けが正しいか

### 表示形式

マークダウンテーブルでプレビュー表示する:

```
| 大項目 | 中項目 | 小項目 | 工数(h) | 売値工数(h) | 備考 |
```

合計行も表示する。ユーザーに確認を求め、調整があればこの段階で反映する。

## Phase 3: CSV出力

ユーザーが項目を承認したら、CSVファイルを生成する。

### CSV構成

```
（空行）
システム概算見積り
"※ マークアップ・フロントエンドのみ。企画・デザイン・バックエンドシステム等は除く。"
（空行）
作成者,{作成者名}
確認者,{確認者名}
最終更新日時,{YYYY/MM/DD}
（空行）
初期開発費
（空行）
社内費用
大項目,中項目,小項目,工数(人時),売値工数(人時),備考,参考情報
{データ行...}
（空行）
,,,合計,{売値合計}
（空行）
備考
"※ 本見積もりは{精度レベル}です。振れ幅: {振れ幅}"
"※ フロントエンド（マークアップ・JavaScript実装）のみの見積もりです。"
（空行）
覚え書き
"※ 工数: 実際の作業時間。売値工数: スターターキット等の流用がない場合のゼロベース工数。"
"※ 工数目安は中堅プレイヤー（2〜3年目）想定です。"
```

### CSV生成ルール

- **UTF-8 BOM付き**（`\xEF\xBB\xBF` を先頭に付与）
- カンマを含むフィールドはダブルクォートで囲む
- 改行を含むフィールドもダブルクォートで囲む
- デフォルトファイル名: `{プロジェクト名}_estimation_{YYYYMMDD}.csv`
- 保存先をユーザーに確認してから Bash ツールで書き出す

### CSV書き出し方法

Python スクリプトを使って BOM 付き UTF-8 で書き出す:

```python
python3 -c "
import csv, io, sys
bom = '\ufeff'
data = [...]  # CSV行データ
with open('output.csv', 'w', encoding='utf-8', newline='') as f:
    f.write(bom)
    writer = csv.writer(f, quoting=csv.QUOTE_MINIMAL)
    for row in data:
        writer.writerow(row)
"
```

## Phase 4: 調整

CSV出力後、ユーザーから以下の調整依頼を受け付ける:

- 項目の追加・削除
- 工数の増減
- 備考の修正
- 新しいページの追加

調整後は Phase 2 のプレビューを再表示し、確認後に CSV を再出力する。
