# React コーディングルール

Reactコンポーネントを作成する際に従うルール。

## コンポーネントの基本

- クラスコンポーネントではなく、**関数コンポーネント**を使用する
- 状態管理には**React Hooks**を使用する

## Props の定義

- Props は TypeScript の `interface` または `type` で定義する
- **型名は `Props` で統一**する（`XxxProps` のような命名は不要）
- コンポーネントファイル内でのみ使用する型は export しない

## JSX とロジックの分離

- JSX 内ではロジックを複雑にしない
- 複雑な条件分岐や計算は関数に切り出す
- コンポーネントのロジックは可能な限り**コンポーネントの外に切り出す**

## カスタムフック

- 再利用可能なロジックはカスタムフックに切り出す
- カスタムフックは `use` プレフィックスで命名する
- 1つのカスタムフックは1つの責務に集中する
