# Commit and Push

現在の変更をコミットしてリモートリポジトリにプッシュする。

## 手順

1. `git status` で変更ファイルを確認
2. `git diff` で変更内容を確認
3. `git log` で最近のコミットメッセージのスタイルを確認
4. 変更をステージングしてコミット
5. `git push` でリモートにプッシュ
6. プッシュ完了後、`git status` で状態を確認

## Commit Message Type

| Type | 説明 |
|------|------|
| `fix` | バグの修正パッチ |
| `feat` | 新機能の追加 |
| `docs` | ドキュメントの変更 |
| `style` | コードのスタイルの変更（セミコロンの追加や改行の修正など） |
| `refactor` | コードのリファクタリング（バグの修正や機能の追加は含まない） |
| `perf` | パフォーマンスの向上 |
| `test` | テストの追加・修正 |
| `build` | ビルドシステムの変更または依存関係のアップデート |
| `ci` | CI/CD の設定の変更 |
| `chore` | その他の変更 |

## コミット粒度のガイドライン

- 1つのコミットには1つの論理的な変更のみを含める
- 複数の異なる機能や修正を1つのコミットにまとめない
- 変更が大きい場合は、関連する部分ごとに分割してコミットする
- 各コミットは単独でレビュー・理解できる単位にする

## 注意事項

- コミット前に必ず変更内容を確認すること
- 暗号化されていない秘密情報を含むファイルはコミットしないこと
- **develop / main ブランチに直接 push しないこと**（必ず feature ブランチからPRを作成する）
