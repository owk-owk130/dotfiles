# TypeScript コーディングルール

TypeScriptコードを作成する際に従うルール。

## 命名規則

- 変数名・関数名は**意味のある名前**を使用する
- 略語は避け、明確な命名を心がける

## Export 方針

- **named export を優先**する（default export は避ける）
- ライブラリの仕様で default export が必須の場合は許容

## 関数の書き方

- **アロー関数**を使用する
- function 宣言は使用しない

## 型定義

- **推論できる型は明示しない**
- 引数の型は必ず定義する
- 関数の return 型は推論に任せる
  - `: void`, `: Promise<void>`, `: string`, `: number`, `: boolean` など自明な型は書かない
  - `apiClient<T>()` のようにジェネリック引数で推論される場合、外側の `: Promise<T>` は不要
  - **例外**: 複雑なオブジェクトリテラルを返す関数や、推論結果が呼び出し側から分かりにくい場合は明示してよい

## パスエイリアス

- path alias が設定されているときは**優先的に使用**する
- **同階層のみ相対パス**を使用する

## データ取得

- **fetch 用の関数を定義**して呼び出す
- axios より**標準 fetch API** を使用する

## その他

- 効率より**コードの可読性を重視**
- 開発のサポートにならないコメントは不要
