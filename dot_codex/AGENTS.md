## コーディング規約

- YAGNI 原則に従う。余計な機能・防御コード・抽象化を追加しない
- DRY を遵守する。重複ロジック・重複定数・コピペを排除する
- npm scripts はプロジェクトで使用されているパッケージマネージャーで実行する
- 新機能・バグ修正は TDD 主体で進める

## レビュー方針

詳細は `~/.claude/codex-review-ruleset.md` を参照。

### 採用する観点

1. **YAGNI / 過剰防御の排除** — 到達し得ないガード、握りつぶす try-catch、未使用の抽象は除去候補
2. **DRY** — 既存の共通実装の再発明、コピペロジック
3. **プロジェクト規約への準拠** — CLAUDE.md / AGENTS.md、各種コーディングルール
4. **de-facto 規約** — リポ内の既存コードの流儀との一貫性
5. **明白な重大バグ** — 型エラー、未定義参照、セキュリティリスク

### 却下するもの

- 過剰防御の追加提案（最重要）
- 好みの nitpick で明確な改善がないもの
- 既存設計と矛盾する提案
- コンテキスト不足による誤解に基づく指摘
